# Ralph Progress Log - v0-platform-mcp Prototype Workflow

## Codebase Patterns
- v0-platform-mcp uses TypeScript 5.4+ with strict mode enabled (ES2022, NodeNext modules)
- ES Modules throughout (type: "module" in package.json)
- Zod schemas validate all MCP tool parameters (see src/types/index.ts)
  - Pattern: Define schema → export type via z.infer
  - Location: All schemas in src/types/index.ts
- Error handling follows V0McpError pattern with categorization (see src/utils/errors.ts)
  - Pattern: ErrorHandler.categorizeError() → returns ToolResult with metadata
  - All errors include: errorType, statusCode, retryable flag
- Winston logging for structured logs (see src/utils/logger.ts)
  - Use logger.info/warn/error with metadata objects
  - Helper: logToolCall(name, success, duration)
- MCP tools registered in src/mcp/tools.ts via V0Tools class
  - Pattern: listTools() returns Tool[] with JSON Schema
  - Pattern: callTool(name, args) validates with Zod → calls service → returns result
- Tests use Jest ^30.0.2 with ts-jest ^29.4.0 (see jest.config.js)
  - Tests go in tests/unit/ mirroring src/ structure
  - Run: npm test (must pass before commit)
- Existing 4 tools: v0_generate_ui, v0_generate_from_image, v0_chat_complete, v0_setup_check
- V0 service abstraction in src/services/v0Service.ts
  - Pattern: Private request() method handles all V0 API calls
  - Pattern: Service methods return Promise<ToolResult>
  - Pattern: try/catch → this.handleError(error) for consistent error handling
- V0 API uses Platform API (POST /chats endpoint)
- Dependencies: @modelcontextprotocol/sdk ^0.5.0, openai ^4.47.1, zod ^3.22.4, winston ^3.17.0

- **Commands:** prepare_prototype_context → generate_prototype → handoff_to_claude_dev
